# 포위 퍼즐 프로젝트 컨텍스트

## 프로젝트 개요
- **장르**: 턴제 전략 퍼즐 (포위 메커닉)
- **본질 기준**: 이 게임은 퍼즐 게임이다. 판 읽기, 수순 설계, 공간 제어, 포위 메커닉 이해를 우선한다.
- **플랫폼/표현 기준**: HTML 기반 2D 게임이다. 모든 기믹과 연출은 HTML 2D에서 명확하게 보이고 조작 가능해야 한다.
- **최적화 기준**: 새 기믹이나 표현을 검토할 때는 항상 성능/복잡도 대비 가치, 렌더링 비용, 계산량을 함께 본다.
- **현재 버전**: v40 (내구도 1 전환)
- **프로토타입**: HTML 기반 (`포위퍼즐-v40.html`, `포위퍼즐-v41.html`)
- **작업 경로**: `~/Desktop/game-planning/포위퍼즐-v40/`

## 핵심 게임 규칙 (v40 기준)
- **배치**: 빈칸 전용, 내구도 1 (반격 1발 = 파괴)
- **포획**: 영역 빈칸 0(완전 밀폐) → 내 화살 최대 매칭(블록·적 좌상단 순) — 벽당 1킬, 미조준 적 생존
- **반격(combat 레벨)**: 밀폐 영역의 적 전원(죽는 적 포함)이 최대 매칭으로 벽에 1타씩
- **확산**: 예고(announced) 칸이 유효하면 그 칸, 무효면 최악 가정으로 비자살 프런티어 전체 분기
- **판정(v37 유예)**: 확산 레벨은 마지막 돌 후 적 턴 1회까지 실행 후 판정. 정지 레벨은 즉시 판정.

## 변경 작업 원칙
- 기믹/규칙/초반 템포를 바꾸는 작업은 구현 전에 반드시 용어 정의와 기준선을 먼저 확정한다.
- 기존 체감에서 절대 안 깨야 하는 항목(예: `내 턴 끝나면 적이 바로 둔다`)은 먼저 문장으로 고정하고, 실험 중에도 유지 여부를 반복 확인한다.
- baseline 파일과 experiment 파일을 섞지 않는다. 실험은 반드시 복사본/별도 파일에서만 진행한다.
- 한 번에 한 축만 바꾼다. 예: 렌더링만, 확산 판정만, 레벨 1만.
- 변경 전 확인은 `change-checklist.md` 템플릿을 사용한다.

## 레벨 구조
- `solver-v40.js` — 적대적(worst-case) 솔버. 레벨 검증 도구
  - 실행: `node solver-v40.js [레벨번호] [--budget N] [--time S]`
  - 예: `node solver-v40.js 12,13 --budget 4`
- `budget-scan.js` / `budget-scan2.js` — 예산(난이도) 범위 스캔
- `find-lv14h.js`, `find-new-levels.js` 등 — 신규 레벨 탐색 도구
- `level-lab-v2.html` — 레벨 설계 & 검증 GUI 도구
- `레벨-감정곡선.html` — 레벨 난이도 감정곡선 시각화

## 현재 방향 (승인된 결정)
- 레벨 1–9: 단축 방향
- 전투(combat) 관련 레벨: 확대 방향
- BGM: 다크·위기감 있는 빠른 템포로 개편

## 레벨 데이터 형식
```js
{ board: N, enemies: [[x,y]], walls: [[x,y,hp]], schedule: [...], spread: bool, combat: bool, min: N }
```
- `board`: 보드 크기 (N×N)
- `enemies`: 적 위치
- `walls`: 벽 위치 + 내구도
- `schedule`: 확산 스케줄 (돌 수 배열)
- `spread`: 확산 레벨 여부
- `combat`: 반격 레벨 여부
- `min`: 클리어에 필요한 최소 돌 수 (예산)
