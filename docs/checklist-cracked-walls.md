# 변경 전 확인 — 금 간 벽 (파괴 가능 경계벽)

> change-checklist.md 템플릿 기반. 작성: 2026-07-20 (Director 스펙 확정 — 파벽자(특수 적)안과 레드팀/블루팀 비교 후 벽 주도안 채택)

## 짧은 실전 버전

```text
1. 바꾸는 대상: 경계벽에 "금 간 벽" 타입 추가 — 적만 부술 수 있는 1히트 가변 지형
2. 핵심 용어 정의:
   - 금 간 벽 = crackedRightWalls/crackedDownWalls. 부서지기 전엔 일반 경계벽과 동일하게 전부 차단.
   - 벽 공격 = 적 턴에 "확산 대신" 인접 금 간 벽 1개 파괴 (이동·확산 없음, 그 턴의 유일한 행동).
   - 대상 결정론 = 인접(벽 양쪽 칸 중 하나가 적) intact 금 간 벽 중 좌상단 우선 (y→x→r<d).
   - 예고 = 실제: 다음 턴 부술 벽에 마커 표시. 공격 턴은 PRNG 소비 없음 (시드 재현성 — 툴과 소비 순서 동일 필수).
3. 절대 안 깨야 하는 기준선:
   - 금 간 벽 없는 레벨(기존 L1~L10 전부)의 min·예고봇·시드 결과 불변 (L9 노드 수 동일까지)
   - 일반 경계벽은 계속 불가침 — "벽은 안전" 문법 유지. 내 돌은 벽을 못 부순다 (비대칭).
   - 예고 = 실제 규칙, 내 턴 끝나면 적이 바로 둔다
4. 이번에 바꾸는 축 1개: 경계벽 타입 + 적 턴 행동 우선순위 (셀 인코딩·볼리 규칙은 무변경)
5. 완료 기준:
   - L1~L10 회귀 전수 일치
   - 금 간 벽 프로토: "시한부 주머니" 판에서 [예고 → 확산 정지 → 벽 파괴 → 봉쇄 붕괴]가 게임·솔버 동일 재현
   - 미끼 동작 확인: 금 간 벽 인접 blob은 확산을 멈춘다 (기회비용)
   - 잠금 연동: 밀폐 생존(잠금) 적의 영역이 벽 파괴로 열리면 잠금 해제
```

## 확정 스펙 (Director, 2026-07-20)

| 규칙 | 내용 |
|------|------|
| 데이터 | `crackedRightWalls: [[x,y]]`, `crackedDownWalls: [[x,y]]` — 미러링은 일반 경계벽과 동일 변환 |
| 파괴 | 1히트 즉시 파괴. **적만 가능** (플레이어는 어떤 벽도 못 부숨) |
| 행동 경제 | 인접 blob이 있으면 그 적 턴은 **확산 대신 공격** (벽 공격이 확산보다 우선) — 금 간 벽 = 미끼로도 작동 |
| 상한 | 판당 금 간 벽 ≤ 3 (위상 상태 2³ — 솔버 검증선 유지) |
| 데뷔 | 고수 구간 (L71+). 초보·하수 금지 |
| 확장 여지 | "파벽자"(금 간 벽 연속 파괴 상위종 적)는 초고수용 후속 카드로 보류 |

## 솔버 설계 (워커)

- 상태 = (보드, 균열 벽 intact 비트마스크). ctx(인접 리스트)는 마스크별 캐시 (≤8개).
- 적 턴: `attackTarget(b, mask)` ≥ 0 이면 결정론 분기 1개 (마스크 갱신 + 영역 잠금 해제 검사), 아니면 기존 확산 분기.
- 공격 턴은 PRNG 소비 없음 — solveWithSeed·게임 computeNextAction 소비 순서 일치.
- memo 키에 마스크 포함.
