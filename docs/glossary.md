# 포위퍼즐 용어집 (정본)

> 마지막 업데이트: 2026-07-21
> 목적: 같은 개념을 여러 단어로 부르는 혼용을 막는다. 문서·주석·설명·UI는 이 표의 **정본 용어**만 쓴다.
> 데이터 필드명(코드 스키마)은 영어 그대로 고정이며, 사람 대면 서술에서는 대응하는 한글 정본을 쓴다.
> 과거 기록(결정 이력의 지난 항목, 게임 규칙 헤더의 v-히스토리 줄)은 당시 표현을 보존한다.

## 0. 가장 중요한 구분 — "내블록" vs "경계벽" (Director 지시, 2026-07-20)

이 게임에는 성격이 다른 두 종류의 막는 것이 있다. **절대 섞어 부르지 않는다.**

| 정본 용어 | 뜻 | 화살 발사 | 데이터 | 쓰지 말 것 |
|-----------|-----|----------|--------|-----------|
| **내블록** | 플레이어가 빈칸에 놓는 것. HP 1. 밀폐 순간 화살을 쏴 적을 처치하고, 전투 시 적의 반격 대상. | ✅ 쏜다(발사 주체) | 런타임 배치(코드 `isBlock`) | **벽**, 돌, 블록 |
| **경계벽(edge wall)** | 레벨에 미리 세워진 지형 담. 적 이동을 막을 뿐 화살은 쏘지 않는다. | ✗ 안 쏜다 | `rightWalls`/`downWalls` | (첫 등장 시 "경계벽(edge wall)", 이후 "경계벽") |

- **"벽"이라는 단어는 경계벽에만 쓴다.** 화살을 쏘는 주체(=내가 놓은 것)를 가리킬 때는 반드시 **내블록**.
- 예: "벽당 1킬" → **"내블록당 1킬"**. "적이 벽에 반격" → **"적이 내블록에 반격"**.
- 예(유지): "3면 경계벽 주머니", "금 간 벽(적만 부수는 경계벽)" — 이 벽은 그대로.

## 1. 핵심 개념

| 정본 용어 | 뜻 | 쓰지 말 것 | 비고 |
|-----------|-----|-----------|------|
| **포위** | 게임 장르·상위 개념(적을 둘러싸 제압). 제목/소개용. | — | 규칙 서술엔 잘 안 씀. |
| **밀폐** | *상태*: 적 영역의 빈칸이 0(완전히 막힘). | 봉쇄, 봉인 | "밀폐되면 포획 발동". |
| **포획** | *사건*: 밀폐 성립 → 내블록이 화살을 쏴 적 처치. | 봉인, 봉쇄 | "포획 발동", "포획 화살". |
| **확산** | 적 턴마다 적이 빈 칸 쪽으로 퍼짐. | 퍼짐, 추격 | 데이터 `spread`. |
| **예고** | 적이 다음에 확산·파괴할 칸을 미리 보여주는 표시. | 텔레그래프(한글 시) | 코드/영문 telegraph. |
| **예고봇** | 예고만 보고 한 수 앞을 두는 검증 봇(직관성 지표). | telegraph bot(한글 시) | "예고봇 100%". |
| **자충** | 갈 곳이 자기 영역의 마지막 빈칸뿐이라 억지로 들어가 자멸하는 확산. | 자살(확산 문맥) | |
| **파동** | 자충 시 적이 쏘는 방출 효과. | — | |
| **반격** | 전투 속성 적이 밀폐 순간 내블록을 되받아치는 공격. | — | |
| **볼리(volley)** | 한 번의 포획에서 오가는 화살 한 라운드(내 화살+반격). | 라운드(혼용 주의) | hp2는 볼리 라운드당 1타. |
| **겹 벗기기** | 포획으로 죽은 칸이 열려 재밀폐로 안쪽 적을 다시 잡는 것. | — | |

## 2. 적 관련

| 정본 용어 | 뜻 | 데이터 | 비고 |
|-----------|-----|--------|------|
| **적** | 처치 대상. `{hp, combat}` 개체. | `enemies:[{x,y,hp?,combat?}]` | HUD "적". |
| **적 HP** | 적이 버티는 피격 횟수(1 또는 2). | `hp` | "적 HP2". 내블록 HP(항상 1)와 구분. |
| **전투 속성** | 그 적이 반격하는지 여부. | `combat`(적 단위) | 레벨 `combat`은 기본값. |
| **혈통 상속** | 확산 자식이 본체의 전투 속성을 물려받음(혼합 인접은 전투 우선). | — | 자식 HP는 항상 1. |
| **잠금** | 볼리 후 생존한 밀폐 적이 재발사 못 하는 상태(영역 재개방 시 해제). | 셀 인코딩 bit2 | |

## 3. 벽·오브젝트 (0절 참조)

| 정본 용어 | 뜻 | 데이터 | 비고 |
|-----------|-----|--------|------|
| **내블록** | 0절. 플레이어가 놓는 것, 화살 주체, HP 1. | 런타임 배치 | **"벽"으로 부르지 않는다.** |
| **경계벽(edge wall)** | 프리셋 지형 담(불가침, 화살 안 쏨). | `rightWalls`/`downWalls` | "벽"은 여기에만. |
| **금 간 벽** | 적만 1히트로 부수는 경계벽(고수 L71+). 내블록·플레이어는 못 부숨(비대칭). | `crackedRightWalls`/`crackedDownWalls` | 호박색 균열 표시. |
| **셀 벽** | 칸 자체를 점유하는 벽(현재 미사용). | `walls` | |
| **회색칸** | 시각 힌트 전용 칸 표시 — "여기 두면 턴 낭비". 배치·확산·포획 규칙에 관여하지 않음(적이 확산 상한 안에 절대 도달 못 하는 칸). | `grayCells` | 솔버·카탈로그·검증 무관(2026-07-21). |

## 4. 수치·설정

| 정본 용어 | 뜻 | 데이터 | 비고 |
|-----------|-----|--------|------|
| **예산** | 그 레벨에서 클리어에 쓸 수 있는 내블록 수. | `min`/`setBudget` | 사람 대면 "예산". |
| **HP** | 내블록·적이 버티는 피격 횟수. 내블록은 항상 1. | `hp` | "내구도"라 쓰지 않는다(Director 2026-07-19). |
| **미러링 / 방향 고정** | 재도전 시 판 반전(다양성). 끄면 고정. | `mirror`(false=고정) | |
| **시드** | 확산 경로를 고정·검증하는 난수 씨앗. | `seeds[]` | |
| **스케줄** | 확산 타이밍(돌 수 배열). | `schedule[]` | |
| **저작판** | 기믹 데뷔·관문·각인·보스. 시드 1 + mirror:false. | — | 전체 ~20%. |
| **채움판** | 커브의 나머지. 구간 연동 시드 풀. | — | 전체 ~80%. |

## 5. 판정·레벨 유형

| 정본 용어 | 뜻 | 비고 |
|-----------|-----|------|
| **전투 레벨** | 반격이 기본 켜진 레벨. | `combat: true`. combat 레벨(한글 시) 대신 사용. |
| **확산 레벨** | 확산은 있고 반격 없는 레벨. | `combat:false, spread:true`. |
| **정지 레벨** | 확산 없는 레벨(즉시 판정). | `spread:false`. |
| **유예 판정(v37)** | 돌 소진 후에도 적 턴(자충 포함)까지 실행 후 판정. | 확산 레벨. |
| **즉시 판정** | 정지 레벨은 마지막 돌 직후 판정. | |
| **클리어/실패** | 클리어=적 전멸 / 실패=예산 소진 후 적 잔존. | |

## 6. 사용 규칙
- 문서·주석·설명·UI는 위 정본만 쓴다. 이명이 필요하면 "정본(이명)"으로 병기 후 정본만.
- **핵심: 화살 주체·플레이어가 놓는 것 = "내블록". "벽"은 경계벽 전용.**
- 데이터 필드명(`min`, `spread`, `combat`, `rightWalls`, `hp` 등)은 영어 그대로. 사람 대면 문장은 한글 정본으로 푼다.
- 레벨 `tag`(감정/테마 라벨, 예: "추격 봉인")는 자유 표현 허용. 규칙 서술은 정본.
- 과거 기록(지난 결정 이력, 규칙 헤더 v-히스토리)은 당시 표현 보존 — 소급 치환하지 않는다.
- 새 개념은 이 표에 먼저 등재하고 나서 코드/문서에 쓴다.
