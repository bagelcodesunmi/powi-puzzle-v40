## 커브 설계 (승인 방향 + 불변 규칙 반영)

- **불변 규칙: 보드판은 절대 줄어들지 않는다** (Director, 2026-07-18). 보드 열: 3·3·4·4·5·5·5·5·5·5 — 단조 증가.
  전투 도입(L6)에서 보드를 되돌리는 대신 **벽 프리셋으로 국소 난이도를 낮춰** 가르친다.
- **1부 (L1~L5, 비전투)**: 포위 기본기. 보드 3→4→5 확장, 예산 계단 1·2·3·4·3.
- **2부 (L6~L10, 전투 · 전부 5×5)**: L6~L7은 **1부에서 익힌 구조를 5×5에 이식 + 반격 규칙** — 새 규칙만 학습.
  L8은 L5와 완전히 같은 판("같은 판인데 이번엔 반격한다"), L9는 구 주머니 판에서 벽 하나 제거(자립),
  L10은 L9와 같은 벽에 적만 한 칸 위(공간 확대 = 난이도 상승) — 벽/적 배치의 연속 변형으로 커브를 만든다.
  (2026-07-21: L5·L8이 복도 판으로 교체되어 L8→L9의 "벽 제거" 시각 연결은 끊김 — L9는 독립 판으로 유지.)
- 감정 흐름: 도파민(1) → 학습(2·3) → 도파민(4) → 학습(5) → 전환점(6, 전투 등장) → 학습(7) → 도파민(8) → 전환점(9, 승률 52%) → 스트레스(10, 승률 30%).

## 검증 기록 (2026-07-18, §14 체크리스트 — 5×5 재설계 후 재검증 · L5/L8은 2026-07-21 교체판 기준)

- **① 최악 확산 min**: L5~L10 전부 적대 솔버로 min 확정 (L5:3, L6:1, L7:2, L8:3, L9:4, L10:5).
- **② 정확히 N턴**: L5·L7·L8 — min−1 예산 우호 확산 5,000판 0승 확인. L9·L10은 분산형(우호 시 더 빨리 깨질 수 있음)으로 의도.
- **③ 시드×미러**: 각 레벨 시드 풀 전수 solveWithSeed PASS (반전 조합 포함). L5 [500002~4], L6 [600001~2], L7 [700001~2], L8 [800002~4], L9 [900001~4], L10 [1000001~4].
- **L5·L8 교체판 재검증 (2026-07-21, verify-level.js)**: 두 판 모두 min 3 확정 · 예고봇 100%(2,000판, median 3) ·
  예산 2 우호 확산 5,000판 0승(정확히 3턴) · 시드 3개 PASS(미러 반전 포함). 게임 로드(localhost) 상태 확인 — 콘솔 에러 0.
- **④ 실제 플레이**: 브라우저(localhost)에서 L6(1탭)·L7(2턴)·L8(3턴, 미러 반전 로드)·L9(4턴)·L10(5턴, "전체 클리어!") 실플레이 클리어 확인. 구 4×4 L9 시절 실패→탭 재시도(같은 시드)→클리어 흐름도 확인. 콘솔 에러 0.
- **§14-2 시드-파티클 한계 해소 (2026-07-20)**: L9·L10은 중간 포획 가능 판이라 과거엔 파티클이 시드 PRNG를 소비해 실제 확산 경로가 검증과 어긋날 수 있었음. 파티클 랜덤 분리(연출 전용 `fxRng`) 완료로 로직 `prng`는 확산·미러링만 소비 → **시드별 확산 경로가 검증 도구와 항상 일치**. L1~L8은 파티클 소비가 없어 회귀 없음.
