## 짧은 실전 버전

```text
1. 바꾸는 대상: 적 표현 — 레벨 전역 속성(combat 플래그, 즉사)에서 적 개체 속성({hp, combat})으로
2. 핵심 용어 정의:
   - 본체 = levels.json enemies에 명시된 시작 적. 확산 칸은 본체가 아님.
   - 전투 속성 = 밀폐 볼리에서 반격에 참여하는 혈통 속성. 본체에서 확산 계열로 상속.
   - hp2 = 본체 한정 HP 2. 확산 자식은 항상 hp1 (Director 확정).
   - 연타 = 볼리에서 아직 안 쏜 벽이, 피격 후 생존한 적(hp2→1)에 인접해 있으면 이어서 1발 더.
3. 절대 안 깨야 하는 기준선:
   - 기존 L1~L10의 min·예고봇 승률·시드 검증 결과가 1도 안 변한다 (hp1 전용 판에서 신규 규칙은 발동 불가 —
     최대 매칭 성질상 hp1 생존 적 곁엔 안 쏜 벽이 존재할 수 없음을 근거로 하되, 반드시 실측 회귀로 재확인)
   - 내 턴이 끝나면 적이 바로 둔다 (턴 템포 불변)
   - 게임·level-lab 워커·솔버·예고봇이 같은 규칙 (워커가 단일 소스, verify-level/catalog는 추출식)
4. 이번에 바꾸는 축 1개: 적 셀 인코딩 + 볼리 규칙 (레벨 데이터·커브는 이번에 안 건드림)
5. 완료 기준:
   - L1~L10 회귀: verify-level로 min/승률/정확N턴/시드 재실행 → 문서 기록값과 일치
   - 카탈로그 스팟체크: 기존 항목 몇 개 재솔브 → 동일 min/유사 승률
   - 신규 규칙 실증: hp2 판 + 혼합(전투/비전투) 판을 브라우저 실플레이로 확인
     (hp2 = 연타 유무 양쪽, 혼합 = 비전투 쪽 무반격·전투 쪽 반격)
```

## 확정 스펙 (Director, 2026-07-19)

| 규칙 | 내용 |
|------|------|
| 데이터 | `enemies: [{x, y, hp?: 1\|2, combat?: bool}]` — 생략 시 hp 1, combat은 레벨 `combat` 플래그를 기본값으로 |
| 전투 상속 | 본체가 전투면 그 확산 계열도 전투, 비전투면 비전투. 혼합 인접(양쪽 blob에 동시 인접한 확산 칸)만 전투 우선 |
| hp2 피격 | 볼리 라운드당 1대미지. 라운드 후 "안 쏜 벽 × 생존 적" 재매칭(연타) 반복 — 벽은 볼리당 1발 |
| hp2 확산 | 자식은 항상 hp1 |
| 반격 | 밀폐 영역의 **전투 속성** 적 전원(죽는 적 포함)이 매칭으로 벽에 1타 — 기존 규칙에서 대상만 속성 필터 |
| 표기 | 전투 적 = 외형 구분(색), hp2 = 숫자 "2" 표시 |

## 셀 인코딩 (게임·워커 공통)

`0 = 빈칸, 양수 = 돌 HP` 유지. 적은 음수 코드:
`-1` 일반(hp1·비전투) · `-2` 전투(hp1) · `-3` hp2·비전투 · `-4` hp2·전투.
판정은 `v < 0` (적), `(-v-1)&1` (전투), `(-v-1)&2` (hp2). hp2 피격 시 -3→-1, -4→-2.

## 첫 시선/터치 · 표현

- 전투 적: 기존 적(빨강)과 다른 색(보라 계열) — "이건 만지면 아프다"가 색으로 읽혀야 함
